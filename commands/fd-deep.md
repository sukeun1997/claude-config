# /fd-deep — 다관점 병렬 설계 탐색

복잡한 FD에 대해 4개 Opus 에이전트를 병렬 실행하여 다양한 관점에서 설계를 탐색한다.

## 사전 조건
- $ARGUMENTS로 대상 FD 번호 필수 (예: `/fd-deep 14`)
- 해당 FD의 Status가 `Design` 또는 `Planned`여야 함

## 절차

1. 대상 FD 파일을 읽는다.
2. 4개 Agent를 **병렬**로 실행 (subagent_type: `architect`, model: `opus`):

   | 에이전트 | 관점 | 분석 초점 |
   |----------|------|-----------|
   | Algorithmic | 알고리즘/데이터 구조 | 시간복잡도, 공간복잡도, 최적 자료구조 |
   | Structural | 아키텍처/설계 패턴 | 모듈 경계, 의존성, 확장성 |
   | Incremental | 점진적 구현 | 단계별 마이그레이션, 하위호환, 롤백 |
   | Environmental | 인프라/운영 | 배포, 모니터링, 장애 시나리오 |

3. 각 에이전트의 분석 결과를 종합하여:
   - 합의 사항 (모든 관점에서 동의)
   - 트레이드오프 (관점별 차이)
   - 권장 접근법

4. FD 파일의 `Considered Solutions` 섹션에 결과를 반영한다.
5. Status를 `Design`으로 업데이트한다.

## 규칙
- 각 에이전트는 독립적으로 FD를 분석 (상호 의존 없음)
- 결과 통합은 메인 에이전트가 수행
- 최종 결정은 사용자에게 위임 (자동 선택 금지)
