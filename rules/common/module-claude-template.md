# 모듈별 CLAUDE.md 템플릿

새 프로젝트/모듈에서 CLAUDE.md 생성 시 이 템플릿을 참조한다.

## 생성 기준

| 모듈 유형 | 생성 필요성 | 이유 |
|-----------|------------|------|
| 도메인 모듈 | **필수** | 비즈니스 불변식, 도메인 규칙 |
| 공통/인프라 모듈 | **필수** | 변경 프로토콜, 영향 범위 |
| 클라이언트 모듈 | 선택 | 특이사항 있을 때만 |
| 앱 모듈 (xxx-app) | 선택 | 배포/설정 특이사항 있을 때만 |

## 필수 섹션

```markdown
# {모듈명} 모듈 규칙

## 핵심 불변식
- 이 모듈에서 절대 위반해서는 안 되는 비즈니스/기술 규칙
- 예: "동일 loanId에 활성 Invoice 최대 1개"

## 변경 프로토콜
- 이 모듈의 코드를 수정할 때 반드시 따라야 하는 절차
- 예: "Avro 스키마 변경 전 Producer/Consumer 매핑 확인"

## 테스트 특이사항
- 이 모듈의 테스트에서 주의할 패턴/설정
- 예: "EmbeddedKafka는 @Autowired lateinit var로 주입"
```

## 선택 섹션

- **아키텍처 패턴**: Hexagonal, CQRS 등 적용된 패턴 명시
- **Consumer/Producer 구조**: Kafka 이벤트 흐름
- **외부 의존성**: 다른 모듈/서비스와의 연동 주의사항
- **EOD/배치 처리**: 일마감, 스케줄러 관련 규칙

## 작성 원칙

- **코드에서 읽을 수 있는 것은 쓰지 않는다**: 클래스 목록, 디렉토리 구조는 AGENTS.md에
- **"왜"를 쓴다**: 규칙의 이유(과거 실수, 제약조건)를 함께 기록
- **검증 가능한 규칙**: "좋은 코드를 작성하라" (X) → "활성 Invoice는 loanId당 1개" (O)
- **200줄 이내**: 간결하게 유지, 상세 내용은 FD 문서나 Notion 참조
