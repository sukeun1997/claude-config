# Layered Context Architecture

3계층 컨텍스트 아키텍처 — Toss "Harness" 패턴 적용.

## 계층 구조

| 계층 | 위치 | 역할 | 예시 |
|------|------|------|------|
| **Global** | `~/.claude/CLAUDE.md`, `~/.claude/rules/` | 모든 프로젝트 공통 규칙 | 코딩 스타일, git 워크플로우, 에이전트 카탈로그 |
| **Project** | `.claude/CLAUDE.md` | 프로젝트 전체 규칙 | 빌드 명령, 테스트 컨벤션, API 설계 |
| **Module** | `모듈/CLAUDE.md` | 모듈 특화 불변식/프로토콜 | 도메인 규칙, 변경 프로토콜, 테스트 특이사항 |

## CLAUDE.md vs AGENTS.md 역할 분리

| 파일 | 역할 | 내용 |
|------|------|------|
| `CLAUDE.md` | **규칙/제약** (What & Why) | 비즈니스 불변식, 변경 프로토콜, 테스트 규칙 |
| `AGENTS.md` | **구조/네비게이션** (Where & How) | 디렉토리 구조, 진입점, 의존성 맵 |

## 규칙 배치 기준

| 적용 범위 | 배치 위치 | 판단 기준 |
|-----------|-----------|-----------|
| 모든 프로젝트 공통 | `~/.claude/rules/common/` | 기술 스택/프로젝트 무관 |
| 프로젝트 전체 | `.claude/CLAUDE.md` | 프로젝트 내 모든 모듈에 적용 |
| 특정 모듈만 | `모듈/CLAUDE.md` | 해당 모듈에서만 의미 있는 규칙 |

## 규칙 승격/강등

- 2개 이상 모듈에서 동일 규칙 반복 → Project 레벨로 **승격**
- Project 규칙이 특정 모듈에만 해당 → Module 레벨로 **강등**
- 2개 이상 프로젝트에서 동일 규칙 → Global 레벨로 **승격**

## 충돌 해결

하위 계층이 상위 계층을 **오버라이드**:
```
Module > Project > Global
```
동일 키의 규칙이 여러 계층에 있으면 가장 구체적인(Module) 규칙을 따른다.
