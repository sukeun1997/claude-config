# Scoring Rubric

각 축 0–5점. 중앙값 3점. 4+는 수월하게 주지 않는다. 1-2점이 보이지 않는 감사는 무른 감사.

---

## 점수 의미 (공통)

| 점수 | 의미 |
|------|------|
| **5** | 동종 업계 상위 5% 수준. 명백히 의도적으로 설계됨. 거의 건드릴 것 없음 |
| **4** | 잘 된 편. 1-2개 micro-issue만 있음. 전반 시스템이 건강 |
| **3** | 평균. 틀린 건 없지만 기억에 남지 않음. 2-3개 명확한 개선 포인트 |
| **2** | 문제가 있음. 시스템이 부재하거나 일관성 부족. 3개+ 이슈 |
| **1** | 시스템 실종 / 원칙 위반 여러 개. 사용자 경험 저해 |
| **0** | 기능 방해 수준. 접근성 위반, 읽을 수 없음, CTA 찾을 수 없음 등 |

---

## Axis 1. Typography — 채점 기준

**5점**: Display + Body 분리, 한국어 혼용 폰트 명시, `font-feature-settings` 활용, 행간·자간 조율, 크기 스케일 4-6단계 의도적

**4점**: 2개+ 폰트 스택, 크기 스케일 일관, 1-2개 micro-issue (예: 숫자 정렬 미처리)

**3점**: 단일 폰트(Inter 류)지만 size/weight 위계는 있음. 한국어 기본 fallback. 크기 종류 2-3개로 단조

**2점**: 제네릭 폰트 + 단일 weight. 제목과 본문 구분 약함. 행간 기본값

**1점**: Arial/Helvetica 직접 지정. 모든 텍스트 동일 스타일. 크기 임의 px 다수

**0점**: 읽을 수 없음 (너무 작거나, 대비 실패로 가독 실패)

---

## Axis 2. Color & Contrast — 채점 기준

**5점**: 의도적 팔레트 (dominant + accent + neutral), semantic 일관, WCAG AA 전체 통과, disabled/placeholder까지 대비 관리

**4점**: 팔레트 일관, AA 통과, semantic 일관. 1-2개 accent 과용 또는 미세 대비 이슈

**3점**: 기본 Tailwind/AntD 팔레트 그대로. semantic은 맞음. 특색 없음

**2점**: primary 색 남용 (모든 버튼 같은 색). 회색 단계 3개 이하. placeholder 대비 미달

**1점**: 보라 그라디언트 on 흰 배경. semantic 혼란 (초록 버튼이 삭제). AA 위반 다수

**0점**: 본문-배경 대비 3:1 미만 (읽기 불가). 색만으로 상태 전달 (색맹 차단)

---

## Axis 3. Spatial — 채점 기준

**5점**: 간격 스케일 6개 이하 고유값, 세로 리듬 일관, 의도된 asymmetry/overlap, 보호 공간

**4점**: 스케일 일관, 리듬 존재. 1-2개 구간 간격 불규칙

**3점**: 기본 스페이싱 토큰 사용 (tailwind p-4, p-6 등). 특색 없지만 깨지지 않음

**2점**: 임의 px 값 다수 (13px, 17px). 섹션 간 간격 불규칙. padding 규칙성 모호

**1점**: 간격 vocab 실종. 카드가 맞닿아 있음 or 과도한 공백. 중앙 정렬 일변도

**0점**: 요소가 겹치거나 잘림. 간격 에러로 가독 실패

---

## Axis 4. Hierarchy — 채점 기준

**5점**: 시선 흐름 명확, primary action 즉시 인지, 상태별 시각 언어 분리, 섹션 헤더 body와 2개 이상 속성 차이

**4점**: 위계 명확. 1-2개 요소가 의도보다 강조됨

**3점**: 위계 있지만 평탄. primary action 찾는 데 1-2초. 빈/에러 상태 디자인 미흡

**2점**: 같은 레벨 카드 5개+ 나열. CTA 2-3개 경쟁. 헤더-본문 구분 약함

**1점**: 위계 실종. 모든 요소가 같은 weight로 경쟁. 인지 부하 높음

**0점**: 사용자가 다음 액션 찾지 못함 (usability test 실패 수준)

---

## Axis 5. Distinctiveness — 채점 기준

**5점**: 3초 기억 단서 있음, 고유 레이아웃/타이포/컬러 중 최소 1개 시그니처, 배경 atmosphere

**4점**: 평균 이상 특색. 기본 템플릿 탈피 시도가 보임

**3점**: 깔끔하지만 평범. 기본 AntD/Tailwind 룩. 경쟁사와 구분 안 됨

**2점**: 템플릿 느낌. 흰 배경 + 회색 카드 + blue accent. 또는 보라 그라디언트

**1점**: 명백한 AI slop. 여러 안티패턴 중첩 (보라 그라디언트 + 3열 아이콘 카드 + Inter 단일)

**0점**: 표절 수준의 일반성. 어느 템플릿인지 즉시 식별 가능

---

## Axis 6. Interaction — 채점 기준

**5점**: 전 상태 설계 (hover/focus/active/disabled/loading/empty/error), 고임팩트 순간 모션, focus ring 커스텀, 키보드 내비 완전

**4점**: 주요 상태 커버, 모션 의도적. 1-2개 상태 누락

**3점**: hover만 처리. loading은 있음. empty/error 디자인 최소

**2점**: 전역 `transition: all`. focus ring 기본값. loading 공백

**1점**: 상태 미설계. focus ring 제거. 모든 hover 동일

**0점**: 키보드 내비 불가능. 클릭 피드백 없음. 접근성 위반

**N/A**: 정적 이미지 입력으로 평가 불가 → 감점 없이 건너뜀

## 증거 형식 표준

각 이슈는 반드시 아래 형식으로 증거 제시:

### 코드 입력
```
증거: src/pages/Dashboard.tsx:42
발견: <Button type="primary"> 3개가 동일 화면에 배치
```

### 스크린샷 입력
```
증거: 스크린샷 상단 히어로 영역 (상단 1/3)
발견: 보라→핑크 그라디언트 + 흰 본문, AI slop 패턴
```

### URL 입력
```
증거: https://example.com/dashboard — 우측 사이드바
발견: 버튼 hover 시 opacity 0.8만 적용, focus ring 부재
```

증거 없는 이슈는 리포트에 포함하지 않는다.

## Severity 매핑

| Severity | 해당 조건 |
|---------|-----------|
| **CRITICAL** | 0점 축, 접근성 위반 (WCAG AA 실패), 프로젝트 CLAUDE.md 규약 위반 |
| **HIGH** | 1점 축, AI slop 2개+ 중첩, 사용자 차단은 아니나 경험 크게 저하 |
| **MEDIUM** | 2점 축, 일관성 이슈, 개선하면 체감 |
| **LOW** | 3점 이하 micro-issue, 취향 영역 |

CRITICAL + HIGH 합계 3개 이하가 "배포 가능" 기준.
