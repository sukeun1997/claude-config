💡 Key Behaviors
Multi-Tool Turns
Model can invoke multiple tools in sequence before emitting a final text response. Each tool result is appended and sent back.
Permission Before Execution
Allow/deny rules are evaluated BEFORE mode check. Bash compound commands are decomposed and each sub-command checked independently.
Streaming
query.ts streams tokens to the terminal in real-time. Oversized tool results (>100K chars) are stored in temporary files.
Compaction
When context window fills, oldest messages are summarized. Full transcript is always preserved on disk for --resume.

⚙ Hook Events Detail
SessionStart
Fires: When a new session begins
  • • Inject today + yesterday daily logs
  • • Load active context for current branch
  • • Check MEMORY.md line count
PreToolUse
Fires: Before each tool execution
  • • frozen.yml file protection
  • • Plan mode routing reminder
  • • Skill auto-routing checks
PostToolUse
Fires: After each tool execution
  • • memory-post-tool.py (memory recording)
  • • edit-tracker.sh (struggle detection)
  • • governance-guard.sh (change surveillance)
  • • skill-usage-tracker.sh
PreCompact
Fires: Before context compaction
  • • Flush critical context to daily log
  • • Update active context file
  • • Preserve key decisions
Stop
Fires: When session is ending
  • • Update active context (handoff)
  • • Detect struggles, suggest self-absorb
  • • Warn if daily log is empty
SessionEnd
Fires: After session closes
  • • Archive daily logs older than 14 days
  • • Move merged branch active contexts
  • • Cleanup temporary files
