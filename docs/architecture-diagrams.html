🔄 Hook Lifecycle — 세션 시작부터 종료까지
각 lifecycle 이벤트에 등록된 훅들의 실행 순서와 역할. sync = 결과 대기, async = 백그라운드, blocking = exit 2로 차단 가능
sequenceDiagram
    actor User as User/CLI
    participant SS as SessionStart
    participant PT as PostToolUse
    participant PR as PreToolUse
    participant PC as PreCompact
    participant UP as UserPromptSubmit
    participant ST as Stop
    participant SE as SessionEnd

    User->>SS: 세션 시작
    Note over SS: session-start.sh
promote-analyzer.sh
    SS-->>User: context 주입

    loop 작업 루프
        User->>PT: 도구 사용
        Note over PT: post-tool.py (*)
edit-tracker.sh (Edit)
governance-guard.sh (Edit)
skill-usage-tracker.sh (Skill)
agent-usage-tracker.sh (Agent)
        User->>PR: Plan 모드 진입 시도
        Note over PR: /feature 스킬 안내
    end

    User->>UP: /clear 입력
    Note over UP: pre-clear-handoff.sh
(HANDOFF.md 작성 강제)
    UP-->>User: exit 2 (차단)

    User->>PC: 컨텍스트 한계
    Note over PC: precompact-path.sh
(active context 긴급 저장)

    User->>ST: 응답 완료
    Note over ST: stop-guard.sh
active-context.sh update

    User->>SE: 세션 종료
    Note over SE: post-tool.py flush (async)
session-end.sh (async)
observer-runner.sh (async)
